# Open Framework + Hosted Runtime Commercial Model

The value proposition is intentionally split:

- **Open framework**: you can inspect and run proof logic locally without paying for hosted execution.
- **Hosted runtime**: you pay for operational convenience, reliability, and scale.

## What you get in the open framework

- Proof/profile contracts
- App contract helpers
- Local proof pack usage
- CLI and helper libraries

## What you get by using hosted Riddle

- Managed browser fleet and runtime uptime
- Durable artifact hosting + retention policy
- Server-preview/build-preview workflows
- Auth-safe sessions and secret handling
- Long-running queue, balance checks, retries, and recovery
- Team workflows and agent handoff/review UX

## Positioning rule

Any feature that requires cloud control plane capability belongs in hosted pricing.
Any feature that enables inspection and reproducible proof semantics belongs in open framework.
